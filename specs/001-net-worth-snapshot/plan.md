# Implementation Plan: Net Worth Snapshot MVP

**Branch**: `001-net-worth-snapshot` | **Date**: 2026-03-16 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-net-worth-snapshot/spec.md`

## Summary

Build a self-hosted personal finance dashboard (Next.js + MongoDB + Docker) that shows
current net worth, tracks account value history via an activity log, and supports a
reconciliation workflow where the user updates account balances manually. Account types:
cash, stocks (manual price entry), crypto (manual price entry), and liabilities.

## Technical Context

**Language/Version**: TypeScript 5 / Node.js 20 (Next.js 16.1.6, React 19)
**Primary Dependencies**: Next.js 16, Tailwind CSS v4, Mongoose, Recharts, bcryptjs, jose
**Storage**: MongoDB 7 (via Mongoose ODM)
**Testing**: Playwright (integration/behavior), Jest (pure utils only)
**Target Platform**: Self-hosted Docker container (Linux/arm64 compatible)
**Project Type**: Full-stack web application (Next.js app router, API routes)
**Performance Goals**: Dashboard loads in under 2s; reconciliation save in under 1s
**Constraints**: Docker + Docker Compose required; all config via `.env`; single user
**Scale/Scope**: Single authenticated user; ~10–50 accounts; years of activity history

## Phase 0 Research Summary

See [research.md](./research.md) for full rationale. Key decisions:

- **Chart library**: Recharts — React-native, SVG, composable, TS-first
- **ODM**: Mongoose — schema validation + TS types over native driver
- **Auth**: Custom JWT (bcryptjs + jose) — simpler than NextAuth for single user
- **Token storage**: httpOnly cookie — prevents XSS token theft
- **Activity model**: Append-only `activity` collection — net worth graph aggregated server-side
- **Docker**: Multi-stage build; mongo:7 service with named volume

## Phase 1 Design Summary

See [data-model.md](./data-model.md) and [contracts/api-routes.md](./contracts/api-routes.md).

**Collections**: `users`, `accounts`, `activity`
**Key derived value**: `currentValue` on Account — recalculated and stored on every save
**Net worth graph**: Server-side aggregation of activity log → one net worth value per
calendar day (latest snapshot per account per day, assets minus liabilities)
